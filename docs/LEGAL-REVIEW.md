# Legal Review Checklist — Takedowns

Every takedown class moves through this list before a packet leaves
the building. The reviewer signs off in the case docket.

## Per-packet

- [ ] Confirmed evidence linked, hashed, and dated.
- [ ] Verdict band is **red** with confidence ≥ 0.85 OR a regulator blacklist hit.
- [ ] At least two independent signals (not just a single user report).
- [ ] No protected-class language in the packet.
- [ ] Recipient is the correct registrar/host per current WHOIS / RDAP.
- [ ] No personal-data fields included unless strictly necessary.
- [ ] Council reviewer named and listed.

## Per-class

| Class                       | Extra review |
|----------------------------|--------------|
| Registrar (gTLD)           | None beyond per-packet. |
| Registrar (ccTLD)          | Local-counsel sign-off recommended. |
| Hosting provider           | Confirm reachable abuse contact. |
| Safe Browsing / SmartScreen| Cross-list cannot conflict with active appeal. |
| DNS blocklist publication  | Council quorum vote required. |
| Coordinated takedown groups| Per-group MOU consulted. |

## Standing rule

If any signal is in dispute the packet does **not** leave. We accept
the cost of a slower takedown over a wrongful one.
